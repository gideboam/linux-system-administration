# Step 1 – Creating the Python Logger Application

A Python script called `logger.py` was created.

Example:

```python
import time

count = 1

while True:
    print(f"Log message {count}", flush=True)
    count += 1
    time.sleep(2)
```

## Explanation

### import time

Imports Python's time module, which provides time-related functions.

It was required because the program needs to pause between log messages.

---

### count = 1

Creates a counter variable.

The variable keeps track of the number of log messages generated.

Example:

```
Log message 1
Log message 2
Log message 3
```

---

### while True

Creates an infinite loop.

The program continues running and generating logs until the container is stopped.

---

### print()

Outputs messages to the terminal.

Docker captures this output and stores it as container logs.

---

### flush=True

Forces Python to immediately send the output.

Without this, Python may buffer messages and delay their appearance in Docker logs.

---

### count += 1

Increases the counter value after every log message.

---

### time.sleep(2)

Pauses execution for 2 seconds before generating the next message.

This prevents the application from generating logs too quickly.

# Step 2 – Creating the Dockerfile

A Dockerfile was created:

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY logger.py .

CMD ["python", "logger.py"]
```

---

# Dockerfile Explanation

## FROM python:3.11-slim

Defines the base image.

Docker uses an existing Python environment instead of installing Python manually.

---

## WORKDIR /app

Creates and moves into the application directory inside the container.

All application files will be stored inside `/app`.

---

## COPY logger.py .

Copies the Python logger application from the host machine into the Docker image.

---

## CMD ["python", "logger.py"]

Defines the command Docker runs when the container starts.

It launches the Python logging application.

---

# Step 3 – Building the Docker Image

The Docker image was created using:

```bash
docker build -t exercise21-logger .
```

## Explanation

- `docker build`  
  Creates an image from a Dockerfile.

- `-t exercise21-logger`  
  Gives the image a custom name.

- `.`  
  Uses the current directory as the build context.

After building, the image appeared:

```bash
docker images
```

Example:

```
exercise21-logger:latest
```

---

# Why Was a Docker Image Created?

The image packages:

- Python runtime
- Application code
- Required environment configuration

into one portable unit.

Instead of manually running Python on every server, the image can be deployed anywhere Docker exists.

---

# Step 4 – Running the Container

The container was started using:

```bash
docker run -d --name logger-container exercise21-logger
```

## Explanation

### docker run

Creates and starts a container.

---

### -d

Runs the container in detached mode.

The application continues running in the background.

---

### --name logger-container

Assigns a readable name to the container.

---

### exercise21-logger

Specifies the Docker image to use.

---

# Step 5 – Checking Running Containers

Command:

```bash
docker ps
```

Purpose:

Displays active containers.

Output showed:

```
logger-container
```

running successfully.

---

# Step 6 – Viewing Container Logs

Command:

```bash
docker logs logger-container
```

Purpose:

Displays all logs generated by the container.

Example output:

```
Log message 1
Log message 2
Log message 3
```

Docker captured the output generated by the Python `print()` function.

---

# Step 7 – Following Live Logs

Command:

```bash
docker logs -f logger-container
```

Purpose:

Continuously displays new logs as they are generated.

The `-f` parameter means:

```
follow
```

It behaves similarly to:

```bash
tail -f logfile
```

in Linux.

To exit:

```
CTRL + C
```

---

# Step 8 – Inspecting Container Information

Command:

```bash
docker inspect logger-container
```

Purpose:

Displays detailed information about the container.

Information provided includes:

- Container ID
- Image used
- Running status
- Startup command
- Environment variables
- Network settings
- Storage information
- Logging configuration

---

# Step 9 – Checking Container Status Only

Command:

```bash
docker inspect -f '{{.State.Status}}' logger-container
```

Output:

```
running
```

This extracts only the container status instead of displaying the entire inspection output.
