# Exercise 8 – Initialize Git Repository

## Overview

This exercise introduces the fundamentals of Git version control by creating and initializing a new Git repository. It demonstrates the basic workflow required to start tracking a project, including repository initialization, creating project documentation, staging files, and making the first commit.

## Objectives

The objectives of this exercise are to:

* Create a new project directory named `sensor-project`
* Initialize a Git repository
* Create a `README.md` file
* Stage files using Git
* Create the first commit
* Verify the repository status and commit history

## Key Concepts Learned

During this exercise, I learned how to:

* Initialize a Git repository using `git init`.
* Understand the purpose of the hidden `.git` directory.
* Distinguish between untracked, staged, and committed files.
* Use `git status` to monitor the state of a repository.
* Stage changes with `git add`.
* Create meaningful commits using `git commit`.
* View commit history using `git log --oneline`.
* Understand that Git commits represent snapshots of a project's history.

## Outcome

At the end of this exercise, a new Git repository named `sensor-project` was successfully created and initialized. A `README.md` file was added, staged, and committed as the repository's first commit. The repository status and commit history were verified to confirm that the working tree was clean and that the initial commit had been successfully recorded.
